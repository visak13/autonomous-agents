**Fix = generic per-node PROMPT principles in `_system`/`_user` (NOT few-shot/rigging, d3-OK):** a gather/search/fetch step is a SOURCE → `depends_on` ALWAYS `[]`; the FINAL step combines+delivers (depends_on the merged ids, more=false); bind the tool whenever the step performs a tool action (EVERY such step, not just the first); never duplicate a prior step. After the fix: 4/4 reliable runs → 3 independent web_search + send_mail join(all 3), node-by-node, ready_wave=[n1,n2,n3], executed peak_in_flight=3; full `run_agentic` naturally selects modular-parallel and reaches the authorer.
